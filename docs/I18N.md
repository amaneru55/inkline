# Inkline 国际化

Inkline 从第一阶段开始支持国际化，避免后期从硬编码文案中迁移。

## 原则

- UI 文案不得硬编码在组件内部。
- 文案 key 需要稳定、语义化，不和具体页面布局绑定过深。
- 默认语言至少包含简体中文和英文。
- 缺失翻译必须有明确 fallback 和开发期提示。
- 日期、时间、数字、文件大小、进度和复数形式使用格式化工具，不手写拼接。

## 目录建议

```text
src/
  i18n/
    index.ts
    locales/
      zh-CN.ts
      en-US.ts
```

## Key 命名

key 按领域组织。

```text
app.title
library.empty.title
library.import.action
reader.toolbar.zoomIn
reader.progress.page
settings.theme.mode.dark
settings.theme.name.inkline
task.status.running
```

## 组件要求

- UI 组件 story 必须覆盖长文本和英文文本，避免布局只适配中文。
- 按钮和紧凑控件必须验证不同语言长度。
- 阅读器 overlay、标题栏、设置页和任务队列都必须使用 i18n。

## 错误文案

错误信息分两层：

- 用户可读文案走 i18n。
- 开发诊断信息可以保留英文技术信息，但不得展示敏感路径、Token 或 Cookie。

## 插件与外部来源

插件提供的文案需要声明 locale。缺失当前语言时按插件声明的默认语言显示，并在 UI 中保持可识别来源。
